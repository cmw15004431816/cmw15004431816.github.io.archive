    寄存器设计思路

    寄存器是一种可以储存二进制数据的逻辑单元，能够完成读取和写入的基本操作。

    我们只需要为寄存器定义一个输入（下面称为active），用以控制是否写入。当该控制输入为1时，接收数据（data）作为输入，并将之存入自己的暂存变量（fourbreg）中。寄存器的输出为其当前所存的数据。具体的实现（以4位寄存器为例，命名该寄存器模块为fourbreg，但请不要与其同名暂存变量混淆）如下：

    module fourbreg(
    input active,
    input [3:0] data,
    output reg [3:0] fourbreg
    );

always @ (active) begin
        fourbreg <= data;
    end

endmodule

    当active为1时，模块fourbreg接受4位输入数据data，将之存入fourbreg中并输出，实现数据暂存的功能。

    思考题

    a.

    从我的system.v文件可以看出，我给出的本次实验的寄存器传输系统的实现是相当繁琐的（足足有450多行），因此有必要采取某些可行的方法来简化我的代码，同时增强代码的可读性。

    一个简单的方法就是采取Verilog宏定义。

    我在实验十一中已经给出Verilog宏定义的基础语法，现在让我们来重温一下：

    `define MACRO_NAME value

    但是，上面的宏定义貌似并不能有效帮助我们简化代码。为此，引入带参数的宏定义，其基础语法如下：

    `define MACRO_NAME(param1, param2, ...) ...

    其中： - MACRO_NAME是宏的名称 - param1、param2...是宏的参数（表） - 最后的...是宏的内容

    节选我的system.v的一部分如下：

    if(SW[1:0] == 2'b00 && SW[3:2] == 2'b00 && SW[8:7] == 2'b00)
    begin
        A = reg1;
        B = reg1;
        reg1 = res;
    end
else if(SW[1:0] == 2'b00 && SW[3:2] == 2'b00 && SW[8:7] == 2'b01)
    begin
        A = reg1;
        B = reg1;
        reg2 = res;
    end
else if(SW[1:0] == 2'b00 && SW[3:2] == 2'b00 && SW[8:7] == 2'b10)
    begin
        A = reg1;
        B = reg1;
        reg3 = res;
    end
else if(SW[1:0] == 2'b00 && SW[3:2] == 2'b00 && SW[8:7] == 2'b11)
    begin
        A = reg1;
        B = reg1;
        reg4 = res;
    end

    采用带有三个参数的宏定义：

    `define REG_ASSIGN(para1, para2, para3) begin \
    A = para1; \
    B = para2; \
    para3 = res; \
end

    则可以将上述繁杂的代码简化如下：

    if(SW[1:0] == 2'b00 && SW[3:2] == 2'b00 && SW[8:7] == 2'b00)
    REG_ASSIGN(reg1, reg1, reg1)
else if(SW[1:0] == 2'b00 && SW[3:2] == 2'b00 && SW[8:7] == 2'b01)
    REG_ASSIGN(reg1, reg1, reg2)
else if(SW[1:0] == 2'b00 && SW[3:2] == 2'b00 && SW[8:7] == 2'b10)
    REG_ASSIGN(reg1, reg1, reg3)
else if(SW[1:0] == 2'b00 && SW[3:2] == 2'b00 && SW[8:7] == 2'b11)
    REG_ASSIGN(reg1, reg1, reg4)

    其余部分可以类似简化。

    通过上述的带有参数的宏定义，我们简化了代码，同时也提高了代码的可读性。

    至于进一步简化代码，则可以考虑使用case语句。这里不再继续展开。

    我们已经看到了宏定义在代码简化和增强代码可读性的巨大作用。当代码较为复杂时，应该多使用宏定义以简化代码。
