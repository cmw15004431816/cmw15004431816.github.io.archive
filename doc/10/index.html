    各模块设计思路

    VGA模块

    VGA模块为实验时直接给出，并非我所设计。但是在此我仍想谈一下设计思路。

    为了达到在top模块中调用VGA并在屏幕上画出一个圆的目的，必须理解VGA的设计思路。

    在我们的实验中，我们采取640 $\times$ 480，60 Hz的显示模式，接入的时钟频率为约25 Mhz。

    Din是我们的输入，用于产生RGB信号，而RGB信号则传到屏幕上用于显示。

    我们只需要产生Din信号，将之传入VGA模块中，在其他输入正确实现的条件下即可通过VGA将有关内容显示在屏幕上。

    其他模块（other_module模块）

    我们整个实验的目的是在屏幕上显示一个圆。如上，VGA用于显示，那么这个other_module模块就是用来产生VGA的Din输入，其中含有的就是一个圆所包含的所有信息。

    考虑圆的要素。

    一个圆，由圆心和半径完全确定。

    我们只需要用开关作为输入，用开关的开闭代表圆心的横纵坐标以及半径即可。

    根据已经给出的other_module定义：

    other_module others(.row(row), .col(col), .rdn(rdn), .pixel(pixel), ...);

    我们只需要向其中添加圆的要素：圆心的横纵坐标以及半径，外加始终信号即可。

    于是，我们定义完整的other_module如下：

    module other_module (
    input clk,
    input rst,
    input [9:0] center_x,//圆心横坐标
    input [8:0] center_y,//圆心纵坐标
    input [7:0] radius,//圆的半径
    input [8:0] row,
    input [9:0] col,
    input rdn,
    output reg [11:0] pixel
);

    在other_module这个模块中，我们只需要写出圆的方程，也就是我们要显示的图案的解析式即可。

    本例中的圆的方程为

    $(col - center_x)^2 + (row - center_y)^2 \leq radius^2$

    但是要考虑到位数限制，位溢出导致的计算错误问题。

    故定义一个足够大的寄存器result，存储$(col - center_x)^2 + (row - center_y)^2$的结果。

    即：

    reg [31:0]result;//定义了32位的result，避免溢出问题
always @(*) begin
    result=(col-center_x)*(col-center_x)+(row-center_y)*(row-center_y);
end

    于是只需要在result <= radius * radius时使得输出pixel为一种颜色（圆的颜色），其余时候为另一种颜色（背景颜色）即可。

    在这里，我的圆为白色（12'h000），背景为黑色（12'hFFF）。

    top模块

    VGA模块为实验时直接给出，并非我所设计。

    我只需完成下面未完成的定义：

    module top (
    input clk,
    input rst,
    input ...,
    output [3:0] R, G, B,
    output HS, VS,
    output ...
);

    我们不再需要其他输出。

    我们需要圆的三个要素：圆心横坐标、圆心纵坐标和圆的半径作为输入。故补全top模块的定义如下：

    module top (
    input clk,
    input rst,
    input [3:0] circle_x,
    input [3:0] circle_y,
    input [3:0] radius,
    output [3:0] R, G, B,
    output HS, VS
);

    由于我们并没有足够的开关，不能像在other_module模块一样，将circle_x，circle_y和radius分别定义为10位，9位和8位的输入。

    在这里，我将它们全部定义为4位输入，然后通过拼接操作构造出10位，9位和8位的结果，输到other_module里。

    具体拼接方法如下：

    reg [9:0] circle_x_1;
reg [8:0] circle_y_1;
reg [9:0] radius_1;

always @ (*) begin
        circle_x_1 = {3'b001, circle_x, 3'b110};
        circle_y_1 = {3'b001, circle_y, 2'b10};
        radius_1 = {3'b001, radius, 3'b110};
    end

    然后接入other_module：

    other_module others(
        .clk(clkdiv[1]),
        .rst(rst),
        .center_x(center_x_1),
        .center_y(center_y_1),
        .radius(radius_1),
        .row(row),
        .col(col),
        .rdn(rdn),
        .pixel(pixel)
    );

    这样我们就实现了top模块。

    上板验证结果以及分析

    首先给出上板验证结果：

    分析上板结果如下：

    从右往左依次：最右边的4个开关用于控制圆心横坐标（左右移动），紧接着的4个开关用于控制圆心纵坐标（上下移动），随后的4个开关用于控制圆的半径（大小），再接下来的1个开关用于控制是否显示（处在下方则显示，上方则不显示），其余的开关在本实验中没有用到。

    控制横坐标的开关：向上推的开关数越多，则圆的位置在屏幕上越靠右；控制纵坐标的开关：向上推的开关数越多，则圆的位置在屏幕上越靠下；控制大小的开关：向上推的开关数越多，则圆越大。

    至此，我们实现了通过输入信号改变圆的位置和大小的实验目的。
